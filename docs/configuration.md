# `pluginregistration.json` configuration

Used by `deploy` and generated by `init`. Not required for `sync`.

## `plugins` section

Defines which assemblies to deploy and which solution receives components.

| Field | Description |
|-------|-------------|
| `assemblyPath` | DLL folder or pattern, e.g. `bin/Release` or `bin/Release/*.dll` |
| `solution` | Solution unique name (optional) |
| `excludePluginSteps` | `true` = upload assembly only, skip steps |

## `stepOverrides`

Key: **step name** or **step GUID** (`Id` from the attribute).

| Field | Description |
|-------|-------------|
| `unSecureConfiguration` | Unsecured step configuration |
| `secureConfiguration` | Secured step configuration |
| `description` | Step description (override; not stored on the attribute) |

Values may use `${VARIABLE_NAME}` placeholders — substituted from environment variables at deploy time:

```json
"unSecureConfiguration": "${API_URL}"
```

Set these variables in your pipeline per environment. The Dataverse connection (`DATAVERSE_*`) is also configured at the pipeline level.

## `customApis` (optional)

**Custom API structure in code is the source of truth.** `customApis` in JSON is for:

- API defined **only in JSON** — requires `createIfMissing: true`
- overriding `displayName` / `description`

| Field | Description |
|-------|-------------|
| `uniqueName` | Custom API unique name (required) |
| `displayName` | Display name override |
| `description` | Description override |
| `pluginTypeName` | Full type name, e.g. `MyNamespace.MyPlugin` |
| `createIfMissing` | `true` = create from JSON if missing in code |
| `isFunction` | OData function (JSON-only definition) |
| `isPrivate` | Private API (JSON-only) |
| `bindingType` | `0` Global, `1` Entity, `2` EntityCollection |
| `boundEntityLogicalName` | Entity for Entity / EntityCollection binding |
| `allowedCustomProcessingStepType` | `0` None, `1` AsyncOnly, `2` SyncAndAsync |
| `requestParameters` | Input parameters (JSON-only) |
| `responseProperties` | Response properties (JSON-only) |

See [custom-api.md](custom-api.md) for attribute-based definitions.
